Aula 2
	CMD: 
		cd backend (pasta do projeto)
		code . (ou abrir visual code)

	Visual studio code: (Local para programar o site)
		Rota / recurso
		Métodos HTTP:
 
		GET: Buscar uma informação do back-end
		POST: Criar uma informação do back-end
		PUT: Alterar uma informação do back-end
		DELETE: Deletar uma informação do back-end
		
		"npm node.js" para iniciar o servidor

	Baixar Insomnia: (Local para testar o site utilizando os métodos HTTP)
		request guarda todos os dados da requisição do usuário
		response retorna uma resposta para o usuário

		* Tipos de parâmetros:
		Query Params: Parâmetros nomeados enviados na rota após "?" 				(Filtos e paginação)
		Router Params: Parâmetros utilizados para identificar recursos 				após "/" no request e ":id" no index.js após 				/users/
 		Request Body: Corpo da requisição, utilizado para criar ou alterar 				recursos

		*Ex de request usando get 
			(http://localhost:3333/users?name=Lucas&idade=25)

		OBS: Para escrever no body do insomnia usar sempre json.

	Index.js:
		Em alterar para "scripts": { "start": "nodemon index.js" },		
		Assim o servidor vai resetar e alterar as informação que for 				mudada no codigo fonte na mesma hora.
		(Se der erro pode funcionar --> npm install -g --force nodemon)
		Agora para iniciar o servidor "npm start" e não mais "npm node.js"

	Instalar Knex.js: (query builder, para banco sql)
		* Bancos de dados:
  		* SQL: MySQL, SQLite, PostgreSQL, Oracle, Microsoft SQL Server 			(Melhor Estrutura)
  		* NoSQL: MongoDB, CouchDB, etc (Bancos não relacionais)
   		* Driver: SELECT * FROM users
   		* Query Builder: table('users').select('*').where()

		"npm install knex"
		"npm install sqlite3"
		
		"npx knex init" (criar knexfile.js)

	Entidades: 
		*ONG
		*Caso(incident)
	Funcionalidades:
		Login e Logout
		Cadastrar, Listar e deletar casos
		Cadastro de ONG
		Listar todos os casos (mobile)
		Entrar em contato com a ong (mobile)
	
	Migrations:
		criar pasta src (colocar index.js, routes.js e pasta database)
		em database criar pasta migrations
		npx knex migrate:make create_ongs
		Criar tabelas em create_ongs.js 
		npx knex migrate:latest (Banco de dados criado)
		
		npx knex migrate:make create_incidents
		npx knex migrate:latest (Banco de dados criado)
		

	Criar pasta controllers:
		OngController.js
		IncidentController.js
		ProfileController.js
		SessionController.js
		
		Para ver no terminal um resultado para testar console.log(count);

	npm install cors:
		Ex: em index -->   app.use(cors({origin: 'http://meuapp.com'}));
		    para definir quem pode acessar a aplicação
